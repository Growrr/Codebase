Incorporate the Codebase semantic grid mixins into your own layout, using your own choice of class names.

The Codebase semantic grid mixins are named similar to the Codebase classic grid classes, except that the col numerals are used as in the example SCSS stylesheet below (this arrangement of mixins would give you the example layout above).

You can also control the media query breakpoints using the Codebase breakpoint mixins, as below.

Mixin Explanation
@mixin grid The container mixin. Sets the max width (from variable $grid-max-width), does centering by auto margin right and left.

If a containing wrapper doesn't have this grid mixin included, then a grid system will expand to whatever the width of the container is, without maxing out at $grid-max-width. E.g. it could expand to the BODY width.
@mixin grid--gutter Gives right and left padding to all immediate child block elements (i.e. columns).
@mixin grid--nested Gives negative right and left margin to the grid container equal to the gutter width, so that a nested grid is flush right and left inside their containing column.
@mixin col Float left. Min height 1px. (Applies padding gutter.)
@mixin col--span(X) Column spanning, as calculated out of $grid-max-cols (default is 12).
@mixin col--push(X) Column pushing to the right, as calculated out of $grid-max-cols (default is 12).
@mixin col--pull(X) Column pulling to the left, as calculated out of $grid-max-cols (default is 12).
@mixin col--offset(X) Column offsetting from the left, as calculated out of $grid-max-cols (default is 12).
@mixin col--centered(X) No float. Instead, centering is accomplished by auto margin right and left. Column spanning is as calculated out of $grid-max-cols (default is 12).
