Popovers

Popovers, lightboxes and modals are essentially the same thing. As of v.1.3 only popovers have been included in Codebase.

If you need your popovers, lightboxes and modals to be styled differently, add your own supplemental class(es) and apply your styles to these (e.g. .popover--lightbox).

Example: The simplest .popover--narrow popover with no header or footer:

Popover content.

Close

All popovers in Codebase slide in from the top and appear flush with the top of the window (but you can move them in by adding a margin (or .deco-m) to the .popover block). The reset-overlay is raised and darkened, and the popover pops over in front of the overlay. They can be dismissed by clicking their *__close 'link' (or button or icon) or by clicking the darkened reset-overlay.

  right: 0;

  height: 0;

  transition: all 0.5s;
  background-color: transparent;
  text-align: center;
  cursor: default;
}

@mixin popover {
  position: relative;
  display: inline-block;
  max-height: 85vh;
  margin-right: auto;
  margin-left: auto;
  bottom: 100vh;
  text-align: left;
  overflow: auto !important;
  transition: all 0.5s;
  img {
    max-height: 70vh !important;
    width: auto !important;
  }
}

@mixin popover--narrow {
  min-width: 300px;
  @include break__sm--up {
    width: 600px;
  }
}

@mixin popover--cover {
  display: block;
  width: 100%;
  min-height: 100%;
  height: 100vh;
  max-height: 100vh !important;
  margin: 0 !important;
}

@mixin popover__wrap--is-open {
  bottom: 0;
  height: auto;
  & > * {
    bottom: auto;
  }
}

@mixin popover__close {
  &:hover {
    cursor: pointer;
  }
}
Classes
// COMPONENT CLASS-BUILDER: POPOVERS
// =================================
// File: codebase/scss/11_component-classes/_popovers.scss

.popover {
  @include popover;
  &--narrow {
    @include popover--narrow;
  }
  &--cover {
    @include popover--cover;
  }
  &__wrap {
    @include popover__wrap;
    &--is-open {
      @include popover__wrap--is-open;
    }
  }
  &__close {
    @include popover__close;
  }
}

                .popover
              
              
                Container. (A unique ID is also required, of your choice – see data-popover="{ID}" below.)

                Opening control (e.g. button)
              
            

            

              
                {ID} = the ID of the popover to be opened
              
            

            

              
                .popover__close
              
              
                Onboard close control
              
            

            

              
                Another close control is already built into the darkened reset-overlay, for "click outside to dismiss".
              
            

          
        


        
Incorporating a Media Object


        

          Codebase popovers come with (almost) no styling of their own – so you can do whatever you like with them. If you require a block header, body and footer then you can easult integrate a Codebase media object.
        


        
<div id="popover-3" class="popover">
  <div class="media">

    <div class="media__header">
      Header
    </div>

    <div class="media__body">
      Body content
    </div>

    <div class="media__footer">
      Footer
      <a class="popover__close float--right">Close</a>
    </div>

  </div><!-- /.media -->
</div><!-- /.popover -->

<button class="btn popover__open" data-popover="popover-3">Show popover 3</button>




        
Responsive popover


        

          If you don't use any layout constraint classes, then you get a responsive popover. Extra-wide images and extra-tall images are scaled down so that the entire image always fits in the browser window, and allows room for a small amount of text (for a title, caption or explanation). This gives you the typical "lightbox" functionality.

          Full cover

           Some browsers (particularly iOS Safari) correctly restrict the z-index of position:absolute elements if they are contained within position:relative elements. So although popovers are intended to pop over in front of the darkened reset-overlay (i.e. popovers have a higher z-index), it cannot do so on these browsers. Thus if you are using position:relative to control your page payout you may find it necessary to place your popovers, etc. later in the DOM, outside of all other elements. (The Codebase layout grid systems do not use position:relative.)
