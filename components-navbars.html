Navbars

The navbar is displayed as a hidden stacked menu at smaller window widths, revealed by clicking a "hamburger" navicon.

Notes:

  1. Dropdown menu behavior only happens on window widths above the navbar media query breakpoint. When the nemubar is collapsed on smaller screens, the child lists are displayed as indented lists within the stacked menu.
  2. On window widths narrower than the navbar breakpoint (i.e. for collapsed navbars), all links are followable (including parent links).
  3. On screens at or wider than the navbar breakpoint (i.e. for navbars as navbars), child menus are hidden by default (using the dropdown utility). Clicking the parent link once will open the dropdown, and clicking the parent link again will follow the parent link.
  4. Clicking anywhere outside the dropdown will dismiss the dropdown. Where no parent link href has been set, clicking the parent 'link' will also dismiss the dropdown.

navbar styling has therefore only been done as far as child menus (i.e. not sub-sub menus, etc).

// The `.navbar` always gets a .wrap() called `.navbar__has-wrap` (this is used to enable the navicon toggle to be absolutely positioned over the menubar on smaller screens).
// If you add `.navbar--fixed` to the menubar, then the wrap will also get class `.navbar__has-wrap--fixed`.

@mixin navbar__has-wrap {
  position: relative;
  width: 100%;
  &--is-fixed {
    position: fixed;
    top: 0;
    width: 100%;
    z-index: $reset-overlay--z-index + 3;
    box-shadow: $deco-box-shadow;
  }
}

@mixin body--navbar-is-fixed {
  padding-top: 48px;
}
