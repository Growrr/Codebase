Components

Codebase components are common HTML/CSS patterns. Some have added functionality from a small jQuery script (e.g. popover, slideshow, tabs).

Components examples have been built first as Pug mixins, in order to be useful in rapid prototyping or style guide composition.

Component Mixins and Class Builders

Almost everything in Codebase, including all components, were created first as a set of SCSS mixins. These were then combined and included in their respective class builder mixins. And these class builders are under the control of a set of true !default variables in codebase/sass/02_default-variables/_default-variables.sass.

So you can toggle component classes for inclusion (true) or exclusion (false) in your advanced theme build by overriding these defaults.

This may be useful for reducing your final CSS file size (by exclusing what you don't need). And if you are combining various frameworks with similar named CSS classes, you can duplicate and edit the Codebase class names, giving them a unique (e.g. prefixed) namespace to avoid clashes.

Components and Media Queries

Except for the navbar (which appears as a navbar at widths above the navbar breakpoint, and collapsed for smaller screens), Codebase components do not involve media queries. If you require components to have different behavior at different screen widths, you can include the component mixins within media queries in your own theme.

For example, you may have a tab-system appear as an accordion for small screens down and as side-tabs on medium screens up.

Component Styling

Codebase components attempt to be as minimally styled as possible. The styling is based on a common set of decoration variables, and is built into the Codebase component mixins.

However, even this minimal styling in the components can be switched off by overriding a set of Component Decoration true !default variables in codebase/sass/02_default-variables/_default-variables.sass. With these set to false, Codebase components are as "style agnostic" as possible, so that you can style them any way you want without CSS bloat.

The Codebase component mixins can easily be included and combined and re-styled in your own theme (see Getting Started: Advanced Setup).

(These variables are also used in the decoration classes.)

Reset Overlay

Several Codebase components (e.g. dropdowns, navbars, popovers and off-canvas sidebar require a "click outside to dismiss" function. This is accomplished by raising a full-page panel (called a reset overlay) to cover the page, and to raise the component to a z-index higher than the panel.

The reset overlay serves as a "button" covering the entire page (except for the active component, which is in ffont of the overlay). All components requiring the reset overlay have reset-functions which are triggered by clicking/tapping the reset overlay.

Usually the reset overlay is transparent. But for popovers it is rendered semi-transparent black.

Dropdowns

In Codebase, dropdown components are a derivative of showhide components, sharing some of the same mixins.

Long content with dropdown__content--nowrap
<div class="dropdown">
  <button class="btn btn--sm dropdown__toggle">Dropdown <i class="fa fa-caret-down"></i></button>
  <div class="dropdown__content dropdown__content--nowrap">Long content with <code>dropdown__content--nowrap</code></div>
</div>
Content


Besides clicking the button again to dismiss the dropdown, you can click anywhere outside the dropdown to dismiss it (i.e. click on a transparent full-screen overlay, which is set at a z-index one unit less than the dropdown).

The .dropdown wrapper ensures that the dropdown content appears flush left of the button. The wrapper has display: inline-block so that you can easily build a row of dropdowns and other inline elements.

However, the downside of the .dropdown wrapper having display: inline-block is that the dropdown content is constrained to the width of the .dropdown__toggle element. But there are various ways around this, which you can apply. One option is to add the modifier class .dropdown__content--nowrap (as used in example 1). Another option is for you to specify your required width for the dropdown content.

    Off-Canvas Sidebars

    The Codebase off-canvas sidebar can be used indepentently, or as a modification to a Codebase grid system.

    If you use the off-canvas sidebar as a modification of a grid system, then instead of setting thecol-[size]--span-* of a sidebar column on your lowest grid setting (xs, sm, md or lg), use the classcol-[size]--off-canvas. You can also specify whether the off-canvas is to the left or right.

    You will also need to add a navicon for toggling the sidebar.

    Be sure to set the off-canvas navicondata-off-canvas="" to point to its respective off-canvas sidebarid="".

    It is possible to have more than one off-canvas sidebar. But only one can be activated at a time.

    Example: see the sidebar on this page at different screen widths.

    <div class="off-canvas-navicon off-canvas-navicon--right" data-off-canvas="sidebar-1">
  <i class="fa fa-bars" aria-hidden="true"></i>
</div>

  <div class="off-canvas__close off-canvas__close--left">
    <i class="fa fa-times" aria-hidden="true"></i>
  </div>

    Notes:

    1. The simple setup consists of two outer block elements .slideshow and .slideshow__inner. The inner block serves as the 'slider trolley''' containing all the .slideshow__item blocks.
    2. The Codebase slideshow adds its own prev/next arrows and pager dots. It calculates the number of dots by counting your slides, i.e. counting your .slideshow__item blocks.
    3. If you require your slideshow to have looping behavior, add class .slideshow--looping to the wrapper. Looping works both forwards and backwards. Hitting the forward arrow on the last slide will send the slideshow to the first slide. Hitting the back arrow on the first slide will send the slideshow to the last slide.
    4. The Slideshow will remain the height of your tallest slide plus the pager height. (The height does not auto-adjust while changing between slides of different heights).
    5. Note: only one instance of the Codebase slidershow component can be used per webpage.
    6. Note: The Codebase slideshow component is not capable of auto-run.

    If you are need a slideshow with more capabilities, I recommend Slick by Ken Wheeler.

    Popovers

    A codebase popover is what is sometimes known as a modal in othe frameworks, or as a lightbox (if the main content is an image and the 'background' is darkened). These are essentially all the same things, so 'popover' has been chosen as the most generic name.

    If you need your popovers, lightboxes and modals to diverge and be styled differently, add your own supplemental class(es) and apply your styles to these (e.g. .popover--lightbox).

    Example: The simplest .popover--narrow popover, with no header or footer:

    Lorem ipsum dolor sit amet usu in odio gubergren conclusionemque.

    Close

    <div class="popover popover--narrow " id="popover-example-1">
  <p>Lorem ipsum dolor sit amet usu in odio gubergren conclusionemque.</p>
  <a class="popover__close float--right">
    Close
  </a>
</div>

<button class="btn popover__open" data-popover="popover-example-1">Popover example 1</button>

    All popovers in Codebase slide in from the top and appear flush with the top of the window (but you can move them in by adding a margin to the .popover block). The reset-overlay is raised and darkened, and the popover pops over in front of the overlay. Popovers can be dismissed by clicking their .popover__close 'link' (or button or icon), or by clicking the darkened reset-overlay.

    Class Explanation
    .popover Container. (A unique ID is also required, of your choice – see data-popover="{ID}" below.)
    By default the popover width is unconstrained – so the width becomes whatever the content needs.
    .popover--narrow Narrow constrained layout (optional)
    .popover--cover Full-page cover layout (optional)
    .popover__open
    data-popover="{ID}"    		 Opening control (e.g. button)
    {ID} = the ID of the popover to be opened
    .popover__close Onboard close control
    Another close control is already built into the darkened reset-overlay, for "click outside to dismiss".

    Responsive popover

    If you don't use any layout constraint classes, then you get a "shrink wrap" popover – the popover will take the with of its content (plus paddings). Extra-wide images and extra-tall images are scaled down so that the entire image always fits in the browser window, and allows room for a small amount of text (for a title, caption or explanation). This gives you the typical "lightbox" functionality.
