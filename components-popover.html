Popovers, lightboxes and modals are essentially the same in Codebase (built off the same mixins and controlled by similar jQuery scripts). Their styles are the same even though their class names are different, and they operate the same way.

All popovers in Codebase slide in from the top and appear flush with the top of the window (but you can move them in by adding a margin (or .deco-margin-top) to the modal body). The reset-overlay is raised and darkened, and the popover (or lightbox, or modal) pops over in front of the overlay. They can be dismissed by clicking their *__close 'link' (or button or icon) or by clicking the darkened reset-overlay.

The code for popovers, lightboxes and modals are essentially the same (but not interchangeable within a particular component).

HTML
<div id="popover-1" class="popover popover--narrow">

  <span class="popover__close float--right"><i class="fa fa-times fa-border"></i></span>

  Popover content

</div><!-- /.popover -->

<button class="btn popover__open" data-popover="popover-1">Show popover 1</button>

Incorporating a Media Object


                

                  Codebase popovers, lightboxes and modals come with (almost) no styling of their own – so you can do whatever you like with them. If you require a block header, body and footer then you can easult integrate a Codebase media object.
                


                
<div id="modal-3" class="modal">
  <div class="media">

    <div class="media__header">
      Header
    </div>

    <div class="media__body">
      Body content
    </div>

    <div class="media__footer">
      Footer
      <a class="lightbox__close float--right">Close</a>
    </div>

  </div><!-- /.media -->
</div><!-- /.modal -->

<button class="btn popover__open" data-modal="modal-3">Show modal 3</button>




              
Responsive Lightbox


              

                If you ddon't use any layout constraint classes, then you get a responsive lightbox (or popover, or modal). Extra-wide images and extra-tall images are scaled down so that the entire image always fits in the browser window, and allows room for a small amount of text (for a title, caption or explanation).

                  Incorporating a grid system
              


              

                  Same as for responsive lightboxes (see above), simply don't use any constraint classes, so that the popover (or modal) takes its width from its content – in this case, a grid system.

(Of course, full-cover popovers can contain a grid, so there is no need to add .popover--contains-grid.)

                   Some browsers (particularly iOS Safari) correctly restrict the z-index of position:absolute elements if they are contained within position:relative elements. So although popovers, lightboxes and modals are intended to appear in front of the darkened reset-overlay (i.e. modals have a higher z-index), it cannot do so on these browsers. Thus if you are using position:relative to control your page payout you may find it necessary to place your popovers, etc. later in the DOM, outside of all other elements. (The Codebase layout grid systems do not use position:relative.)
