Layout

Grid Systems

In Codebase three grid systems have been provided (Classic, Semantic and Flexbox). These all have the same characteristics:

  • 12 columns – you can change this simply by modifying the $grid-max-cols variable in your chosen grid flavour
  • Flexible – column width is set using max-width, not width
  • Responsive – media query breakpoint widths – you can change these in scss/base/_base-breakpoints.scss
  • Pushing/pulling (or in case of the flexbox grid, re-ordering)
  • Offsetting
  • Nesting
  • (single column) centering

These three grid systems all display essentially the same in browsers new enough to cope with flexbox (i.e. above IE9) – the only significant difference being that the flexbox columns are the full height of their containers, not merely the height of their own content.

Grids at different Media Query Breakpoints

The Codebase media query breakpoints are specified by four variables. The breakpoint values can easiy be modified.

The Codebase grid systems can be used without specifying media query breakpoints (so the columns will be displayed the same whatever the window width). But if you use the breakpoint-classes in the Classic grid, or if you wrap your semantic grid mixins or flexbox grid mixins in media queries, then you can control the columnar layout differently at different screen widths.

So, each Codebase grid system can have up to five window-width configurations:

Classic grid Semantic grid Flexbox grid
Multi-column below $break-xs Set columns using .col-- (without breakpoint classes). Use column mixins without wrapping them in breakpoint mixins.
Multi-column above $break-xs Set columns using .col-xs--* Wrap column mixins in @include break__xs-up {...}
Multi-column above $break-sm Set columns using .col-sm--* Wrap column mixins in @include break__sm-up {...}
Multi-column above $break-md Set columns using .col-md--* Wrap column mixins in @include break__md-up {...}
Multi-column above $break-lg Set columns using .col-lg--* Wrap column mixins in @include break__lg-up {...}

Visibility at Different Breakpoints

In addition, layout visibility (mixins and classes) have been provided, for showing or hiding items at different breakpoints.
