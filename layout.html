Layout

Grid Systems

In Codebase four grid systems have been provided:

Layout grid classes

  • Flexbox (default) – since Codebase 2.0, the layout grid is built using flexbox by default.
  • Float (deprecated) – the older float grid system still exists, but it is unavailable because of a$grid-flexbox: true !default toggle. Override this variable if you prefer to use a float grid.

Both flexbox and float grids use the same classes (set them up the same way). The only difference visible in the browser is that in the flexbox grid, all columns within the.grid wrapper are the same height.

Layout grid mixins

(For "semantic" grids.)

  • Flexbox mixins – mixins specific to the flexbox grid are prefixedflexbox-col--
  • Float mixins – mixins specific to the float grid are prefixedfloat-col--

Mixins used by both aregrid,grid--gutter,grid--nested andcol.

Common features of Codebase layout grids

All four layout grid systems share the same$grid- variables for controlling the maximum width, number of columns and the size of the gutter. The gutter is controlled by its own class/mixin, so that you can choose whether or not a particular grid instance has a gutter or not.

The grid systems are flexible (column width is set usingmax-width, notwidth) and responsive (controlled by a set of media query breakpoint widths, which are in turn set by overridable$break- variables.)

All the usual variations are supported:

  • column span
  • column offset
  • column order (flexbox grids only)
  • column push/pull (float grids only)
  • column centering (single column only)

The four Codebase layout grid systems all display essentially the same in browsers new enough to handle flexbox (i.e. above IE9). The only significant difference being that the flexbox columns are the full height of their containers, not merely the height of their own content.

Incomplete grid rows? No problem. Rows can be incomplete (no `.end` or `.omega` classes required).

Surplus grid columns? No problem. Columns will wrap if they extend beyond the grid max width.

Grids at different Media Query Breakpoints

The Codebase media query breakpoint widths are specified by four!default variables.

The four grid systems can be used with or without specifying breakpoints (and with or without using grid breakpoint classes). Without breakpoints, the columns will be displayed the same (% width) whatever the window width.

This means that each grid system can have up to five window-width configurations:

Grid classes Grid mixins
1 Multi-column below$break-xs Set columns using.col--. Use column mixins without wrapping them in breakpoint mixins.
2 Multi-column above$break-xs Set columns using.col-xs--* Wrap column mixins in@include break__xs-up {...}
3 Multi-column above$break-sm Set columns using.col-sm--* Wrap column mixins in@include break__sm-up {...}
4 Multi-column above$break-md Set columns using.col-md--* Wrap column mixins in@include break__md-up {...}

Layout Grid Classes

Since Codebase 2.0, the default layout grid classes use flexbox. You can revert to the older float-based grid system by overriding the $grid-flexbox to false. 

$grid-flexbox: true !default;

.col Float left. Min height 1px. (Applies padding gutter.)
.col--centered No float. Instead, centering is accomplished by auto margin right and left.

                    @mixin grid
                    
                      The container mixin. Sets the max width (from variable code $grid-max-width ), does centering by auto margin right and left. If you have more columns (or column width) than the grid container width, wrapping is handled by flex-wrap: wrap;
                      br
                      br
                      | If a grid container doesn't have the grid mixin included, then a grid system will expand to whatever the width of the container is, without maxing out at $grid-max-width. E.g. it could expand to the BODY width.
                    
                  

                  

                    @mixin grid--gutter
                    Gives right and left padding to all immediate child block elements (i.e. columns).
                  

                  

                    @mixin grid--nested
                    Gives negative right and left margin to the grid container equal to the gutter width, so that a nested grid is flush right and left inside their containing column.
                  

                  

                    @mixin col
                    (Min height 1px.)
                  

                  

                    @mixin flexbox-col--span(X)
                    Column spanning, as calculated out of $grid-max-cols (default is 12).
