Getting Started

Simple Setup

  1. Add codebase.css and codebase.js to your project.
  2. Add Font Awesome and jQuery (either self-hosted a or link to a CDN).
  3. Create your first HTML page, e.g. using the starter example below.
<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge,chrome=1">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>Your Webpage Title</title>
    <link rel="stylesheet" href="path/to/font-awesome.min.css">
    <link rel="stylesheet" href="path/to/codebase/css/codebase.css">
    <link rel="stylesheet" href="path/to/your-theme/css/styles.css">
  </head>
  <body>

    <!-- Get to work -->

    <script src="path/to/jquery.js"></script>
    <script src="path/to/codebase/js/codebase.js"></script>
    <script src="path/to/your-theme/js/scripts.js"></script>
  </body>
</html>

Note that jQuery needs to be sourced first, so that the Codebase jQuery plugin can work.

Advanced Setup

Example project folder structure:

your-project/
├── codebase/
│     ├── (css/) /* Unrequired if you are importing Codebase into your theme */
│     ├── coffee/
│     ├── js/
│     └── sass/
│
├── your-dev/
│     ├── coffee/
│     └── sass/
│
└── your-build/
      ├── css/
      └── js/

Import Codebase into your theme SASS

All Codebase variables have the !default flag. To override any of these, you first declare your your overrides using the same Codebase variable names (but with your preferred values), then import Codebase. Your SASS pre-processor will preferentially use your values instead of the Codebase defaults. 

// YOUR PROJECT BASE SASS FILE
// ===========================

// (1.) Override Codebase `!default` variables here...

// (2.) Import Codebase
@import '../../codebase/scss/codebase';

// (3.) Begin your project styles here...

Whenever you use any of the the Codebase variables within your own variables, you will need to declare these first, before you use them.

For example: if you need to change the navbar breakpoint from $break-xs to $break-sm then you need pre-declare $break-sm in your own variables.

You could copy the entire Codebase default variables file (codebase/scss/02_default-variables/_default-variables.scss) into your theme (remove all the !default flags) and set your chosen variable values there.
