Getting Started

Simple Setup

First, copy the codebase/ folder (containing the css/, js/ and scss/ folders) into your project.

Example simple HTML to get started: 

<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge,chrome=1">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>Your Webpage Title</title>
    <link rel="stylesheet" href="path/to/codebase/css/codebase.css">
    <link rel="stylesheet" href="path/to/your-theme/css/styles.css">
  </head>
  <body>

    <!-- Get to work -->

    <script src="path/to/jquery.js"></script>
    <script src="path/to/codebase/js/codebase.js"></script>
    <script>
      $(document).codebase();
    </script>
    <script src="path/to/your-theme/js/scripts.js"></script>
  </body>
</html>

Note that jQuery needs to be sourced first, so that the Codebase jQuery scrips can work.

Advanced Setup

(1.) SCSS

After copying in the codebase/ folder as above, build your project theme structure, e.g.: 

your-project/
  ├── codebase/
  │     ├── css/
  │     ├── js/
  │     └── scss/
  │
  └── your-theme/
        ├── css/
        ├── js/
        └── scss/

You can import codebase.scss directly into your project base SCSS file.

There are two ways that you can override the Codebase default variables – the choice is yours:

"Non-interventionist" variable overrides

This is where you declare your variable overrides before you import Codebase. All Codebase variables have the !default flag, so if you name your overrides the same as the Codebase variable names, then your SCSS pre-processor will preferentially use your values instead of the Codebase defaults. For example: 

// YOUR PROJECT BASE SCSS FILE
// ===========================

// Put your overrides of Codebase `!default` variables here

// Then import Codebase
@import '../../codebase/scss/codebase';

// Your project styles here
...

This "non-interventionist" override approach has this advantage: you can later easily drop into your project a newer version of Codebase. But the disadvantage is that you can't use any of the the Codebase variables within your variables. (E.g. you couldn't change the navbar breakpoint from $break-xs to $break-sm unless you had previously declared $break-sm. )

"Interventionist" variable overrides

This is where you actually declare your overrides within Codebase, after the default variables. Codebase imports an empty file named _variable-overrides.scss for this purpose: 

@import '01_default-variables/ ...

@import '02_variable-overrides/_variable-overrides.scss

...

The advantage of this "interventionist" approach is that you have full use of everything. The disadvantage is, you must remember to copy or backup your _variable-overrides.scss before you update Codebase.

(2.) JS / jQuery

The Codebase jQuery script is a set of (cb_-prefixed) plugins – each of which you can call – and a "master" (document ready setup) plugin named codebase();.

You will need to call codebase();. The simplest way to do this is: 

<script>
  $(document).codebase();
</script>

Alternatively, you could move that call into your own theme script.
