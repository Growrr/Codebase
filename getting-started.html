Simple Setup

First, copy the codebase/ folder (containing the css/, js/ and scss/ folders) into your project.

Example simple HTML to get started: 

<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge,chrome=1">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>Your Webpage Title</title>
    <link rel="stylesheet" href="path/to/codebase/css/codebase.css">
    <link rel="stylesheet" href="path/to/your-theme/css/styles.css">
  </head>
  <body>

    <!-- Get to work -->

    <script src="path/to/jquery.js"></script>
    <script src="path/to/codebase/js/codebase.js"></script>
    <script>
      $(document).codebase();
    </script>
    <script src="path/to/your-theme/js/scripts.js"></script>
  </body>
</html>

Note that jQuery needs to be sourced first, so that the Codebase jQuery scrips can work.

Advanced Setup

(1.) SCSS

After copying in the codebase/ folder as above, build your project theme structure, e.g.: 

your-project/
  ├── codebase/
  │     ├── css/
  │     ├── js/
  │     └── scss/
  │
  └── your-theme/
        ├── css/
        ├── js/
        └── scss/

You can import Codebase into your project base SCSS file. But before you do that, you may wish to override any/all of the SCSS variables in Codebase. This can easily be done because they are all set as !default variables. To override them, declare or import your overrides (i.e. variables with the same names but your preferred values) before importing Codebase.

So for example, if you have copied the Codebase folder as a sibling folder beside your project theme folder (as I have done for this documentation 'theme'), then you can set up as follows: 

// YOUR PROJECT BASE SCSS FILE
// ===========================

// Put your overrides of Codebase `!default` variables here

// Then import Codebase
@import '../../codebase/scss/codebase';

// Your project styles here
...

(2.) JS / jQuery

The Codebase jQuery script is a set of (cb_-prefixed) plugins – each of which you can call – and a "master" (document ready setup) plugin named codebase();.

You will need to call codebase();. The simplest way to do this is as in the example (here repeated): 

<script>
  $(document).codebase();
</script>

Alternatively, you could move that call into your own theme script.
