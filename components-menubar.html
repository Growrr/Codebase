Notes:

  1. It it not necessary to add the dropdown utility classes into the menubar. (The dropdown mixins have already been applied to the parent list item, the parent link and the child list.)
  2. Dropdown menu behavior only happens above the menubar media query breakpoint. When the nemubar is collapsed on smaller screens, the child lists are displayed as indented lists within the stacked menu.
  3. On screens narrower than the menubar breakpoint (i.e. for collapsed menubars), all links are followable (including parent links).
  4. On screens at or wider than the menubar breakpoint (i.e. for menubars as menubars), child menus are hidden by default (using the dropdown utility). Clicking the parent link once will open the dropdown, and clicking the parent link a second time will follow the parent link. (Clicking anywhere outside the dropdown except the parent link will dismiss the dropdown.)

Menubar styling has therefore only been done as far as child menus (i.e. not sub-sub menus, etc).

Menubars can incorporate dropdown child-menus (using the dropdown utility, same as for button dropdowns). But in menubars this functionality has been added: when displayed as a menubar (i.e. above the menubar breakpoint), dropdown parent links can be clicked once to reveal the dropdown menu and clicked again to follow the parent link. Click outside a revealed dropdown to dismiss it.
