Codebase v.0.7 Docs

Codebase is still in "immature" development. Everything you see here is functional, committed and ready to use in your project — don't be concerned about the version zero-point- status. More components are planned to be added in future.

In developing Codebase, these rules for Codebase styling are being followed.

Currently these docs are simply a single demo page with minimum explanation.

(Note: the docs body background has been set to pale gray so that module instances can be indicated with a white background.)

Grid System

Three flavours of grid have been provided (Classic, Semantic and Flexbox). These all have similar display behaviour:

  • 12 column, but you can change this simply by modifying the $grid-max-columns variable in your chosen grid flavour
  • Flexible – columns set using max-width, not width
  • Responsive – media query breakpoint widths: 768px, 1024px and 1200px – you can modify these in scss/base/_base-breakpoints.scss
  • Push/pullable (or in case of the flexbox grid, reorderable)
  • Offsetable
  • Nestable
  • (single column) centerable

In addition, visibility classes have been provided, for showing/hiding items at different breakpoints.

Note: while all three grids have been included in this Repo instance of Codebase, users are encouraged to @include only their flavour of choice.

Menubar

The menubar is displayed as for Menu (below) at smaller window widths. When displated as a menubar, parent links can be clicked once to reveal the dropdown menu, and clicked again to follow the parent link.

Click outside the dropdown menu to dismiss it (i.e. click on a transparent full-screen overlay, which is set at a z-index one unit less than the dropdown).

So far, styling has only been developed as far as child links (i.e. not sub-sub menus, etc).

You can apply the .current class (rename it if you need to) to either the <li> or the <a> – whichever suits the CMS system you are theming.
