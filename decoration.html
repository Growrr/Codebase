Decoration

For consistency throughout Codebase, some decoration variables have been defined. Where necessary, a few of these variables have been used in the various components.

These same variables have been incoporated into a set of simple decoration classes. These classes can be used to quickly add consistent styling to your HTML directly.

You can modify the decoration variables and classes, and add to them, in accordance with the style guide or visual identity of your website project.

Hat tip:
the idea for Codebase decoration was inspired by BASCSS, Tachyons and the 8-Point Grid.

Element Grid

Since version 2.0 Codebase uses a "soft" 8-point grid system for consistently positioning things relative to each other. (This is also known as an 8px baseline grid or an 8dp square baseline grid.

The Codebase element grid has been deployed a follows: in setting typographic line-heights, margins, and paddings for components, multiples of 8px* have been used. Also by default, the base typography vertical rhythm (line-height) is set at 1.5rem = 24px = 3 × 8 element grid units. (And the layout grid gutters have been set to multiples of 8px.)

[* Device independent pixels, also known as CSS pixels.]

All Codebase margins, paddings and line-heights have been set using $spacing-* variables as in the table below. The base 8px is a !default setting – you can override this simply by changing the value of $spacing-1u.

element grid system can be used in combination with any Codebase layout grid system.
Element Grid Variable Calculation Size
$spacing-1u Base = 8px !default 8px
$spacing-2u Base × 2 16px
$spacing-3u Base × 3 24px
$spacing-4u Base × 4 32px
$spacing-5u Base × 5 40px
$spacing-6u Base × 6 48px
$spacing-7u Base × 7 56px
... ... ...

The $spacing- variables go up to 20×. The decoration classes that use them (margins, paddings) also go up to 20×

[Note: Since Codebase v.1.4, decoration variables/classes named *-sm, *-md etc. have been removed.]

Margins

Element grid units (8px) designated -1u to -20u, plus -zero (for removing unwanted margin).

  • Margins all: m--*
  • Margin top: mt--*
  • Margin right: mr--*
  • Margin bottom: mb--*
  • Margin left: ml--*
  • Margins vertical: mv-* (top and bottom)
  • Margins horizontal: mh-*(left and right)

Examples:

.mr--zero
.mr--1u
.mr--2u
.mr--3u
.mr--4u
.mr--5u
.mr--6u

...

.mr--20u

Horizontal centering by margin:auto

This also sets the element to display:block !important

.mh-auto

Borders

A few simple border styles.

  • Borders all b-*
  • Border top bt-*
  • Border right br-*
  • Border bottom bb-*
  • Border-left bl--*

Examples:

.b--zero
.b--1px-dashed
.b--1px-solid
.b--4px-solid
.b--8px-solid

By default, borders have color #code $color-gray-lighten-2]. Borders can be assigned other colors in the Codebase swatch using b-[color] classes. See colors.

Border enhancements:

.b--radius
.b--circle

Paddings

Element grid units (8px) designated -1u to -20u, plus -zero (for removing unwanted padding).

  • Paddings all: p-*
  • Padding top: pt--*
  • Padding right: pr--*
  • Padding bottom: pb--*
  • Padding left: pl--*
  • Paddings vertical: pv-* (top and bottom)
  • Paddings horizontal: ph-*(left and right)

Examples:

.pr--zero
.pr--1u
.pr--2u
.pr--3u
.pr--4u
.pr--5u
.pr--6u

...

.pr--20u

Block padding

E.g. used for menu items, navbar items, tab labels and table cells. Equal to pv-1u ph-2u

.p--block

Colors

The Codebase swatch

The default Codebase swatch re-purposes the control colors (primary, secondary, warning, error, success) as decoration colors (blue, teal, orange, red, green respectively), and adds black, gray and white. A series of lighter and darker variants of each color have been generated (except for black and white).

You can override and add to these colors in your own theme.

The Codebase swatch displayed as background colors in a table:

$color-white
$color-[color]-lighten-3
$color-[color]-lighten-2
$color-[color]-lighten-1
$color-[color]
$color-[color]-darken-1
$color-[color]-darken-2
$color-[color]-darken-3
$color-black

In each color series there are 7 colors: the base color (middle row above) plus 3 incrementally lighter and 3 incrementally darker.

Applying colors to elements

In addition to the colors (optionally) used in the typography and components, Codebase also has a set of decoration classes for setting colors of text, backgrounds and borders.

Backgrounds bg-[color]
Example .bg--blue-lighten-3

Borders b-[color]
Example .b--orange
Text text-[color]

Example .text--teal-darken-1
